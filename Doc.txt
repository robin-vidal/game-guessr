# MVP PRODUCT STRATEGY

## MVP Objective

Deliver a **fully authenticated multiplayer experience** where users can:

- Log in via OAuth (Using Epita Identity provider if possible)
- Create private rooms
- Invite friends via link
- Play a complete 3-phase round
- Compete in real-time
- See leaderboard results
- Run entirely on Kubernetes with CI/CD

---

## MVP Scope

### Included in MVP

- OAuth login (Authentik + Epita provider)
- Persistent user accounts (linked to OAuth identity)
- Private rooms with invite links
- Host role
- 1 curated game pack (Mario Kart)
- 3-phase gameplay
- Real-time multiplayer sync
- Leaderboard
- Kafka-based scoring
- Redis real-time lobby state
- Kubernetes deployment with Helm
- CI/CD pipeline

## Explicitly Out of Scope (Post-MVP)

- Multiple game packs
- Public rooms (auto-matchmaking)
- Advanced anti-cheat
- Seasonal rankings
- Global leader board
- Advanced analytics dashboards
- ArgoCD GitOps
- Mobile device support

# EPIC A — Authentication & User Management (MVP Critical)

---

## A1 — OAuth Login (Authentik+Epita Integration)

### User Story A1.1 — Login via OAuth

> As a Player
> 
> 
> I want to log in using OAuth
> 
> So that I can securely access multiplayer features.
> 

**Acceptance Criteria:**

- "Login" button redirects to Epita Identity provider
- OAuth Authorization Code Flow
- Backend validates JWT
- User created in database if first login
- Session stored securely
- Logout supported

> As a Player
> 
> 
> I want my identity persisted
> 
> So that I can see my username and results across matches.
> 

**Acceptance Criteria:**

- Unique internal user ID
- Username from OAuth claims
- Display name visible in rooms & leaderboard
- Display picture if Epita provider lets us access it

# Post-Release User Stories

## Phase 3 — Spatial Precision (Completion of Gameplay Loop)

### 1.1 Drop Pin Mechanic

**User Story**

As a competitive player

I want to drop a pin representing my guessed `(x, y, z)` location

So that spatial precision becomes part of the scoring.

**Acceptance Criteria**

- Phase 3 unlocks after Phase 2 completion.
- Player can place and adjust a pin on a 2D top-down map (or controlled 3D interface).
- Submission locks the answer.
- Coordinates are sent to the backend for validation.

---

### 1.2 Euclidean Distance Scoring

**User Story**

As a player

I want my score to be calculated based on distance from the true spawn

So that precision is rewarded proportionally.

**Acceptance Criteria**

- Backend computes 3D Euclidean distance.
- Score range (e.g., 0–5000 points).
- Distance and awarded points are displayed in round results.
- Score is aggregated into the total match score.

---

## 2. Public Matchmaking Enhancements

### 2.1 Dedicated Matchmaking Queue

**User Story**

As a player

I want to enter a matchmaking queue

So that I am automatically grouped with other available players.

**Acceptance Criteria**

- A queue system assigns players to rooms automatically.
- Rooms are created when minimum player threshold is reached.
- Timeout logic handles low-traffic scenarios.
